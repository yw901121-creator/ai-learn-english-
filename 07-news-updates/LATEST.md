## 5 things you absolutely need to know today

### 1) OpenAI launches the GPT-5.6 family (Sol / Terra / Luna)
- **What it is:** Instead of shipping only one flagship model, OpenAI now offers a three-tier product line  
  - **Sol** = top-tier flagship  
  - **Terra** = balanced everyday model  
  - **Luna** = fastest and cheapest  
- **New concept:** `ultra` mode - multiple Agents working in parallel on hard tasks  
- **How to mention it in an interview:**  
  > "Major AI companies now control cost through model tiering and routing. In an education platform, high-volume Q&A can use the lower-cost tier, while difficult explanations can be routed to the flagship tier."

### 2) Anthropic: Claude Fable 5 remains in paid plans
- Fable 5 continues to serve as Anthropic's higher-end capability model  
- Different plans have different quotas and pricing structures (Max/Premium vs Pro)  
- **How to mention it in an interview:** flagship models usually require capacity and cost management, so products should have fallback logic.

### 3) Moonshot releases Kimi K3 (very large, open-source-oriented)
- Reported at roughly the **2.8T** parameter scale, with long context and native vision  
- Weights are expected to open later (watch for developments around 7/27)  
- **Why it matters:** open vs closed competition between China and the U.S. is intensifying, and enterprises have more private deployment options

### 4) Alibaba previews Qwen3.8
- Also positioned around high capability with an upcoming open-weight path  
- **What it means for you:** in interviews, you can say, "I would evaluate whether an open-source model is suitable for private deployment."

### 5) The mainstream architecture terms are unchanged, but more mature: RAG + MCP + Agent
- RAG = knowledge  
- MCP = standard for tool connectivity  
- Agent = execution  
- Education research and products are also combining these three into intelligent tutoring systems

## New terms this week (memorize these)

| New term | Meaning | When to use it |
|---|---|---|
| **Model Tiering** | Layering models by level (flagship / balanced / low-cost) | When discussing cost and routing |
| **ultra (OpenAI)** | Parallel acceleration mode with multiple Agents | When discussing Agentic capability |
| **Open-weight** | Model weights are available for self-hosted deployment | When discussing open-source strategy |
| **Long-horizon Agent** | An Agent that can sustain long, multi-step tasks | When discussing complex workflows |
| **Computer Use** | AI operating a computer interface | When discussing digital workers |

## Impact on "education AI interviews" (key points)

1. **Do not talk only about GPT anymore** - you should be able to compare multiple vendors and open-source options  
2. **Be able to discuss routing** - different teaching tasks should use different model tiers  
3. **Be able to discuss controllability** - the stronger the Agent, the more permissions and auditability matter  
4. **Be able to discuss open-source options** - school clients often ask whether data leaves the country or organization
